Dicas de formatação e extras


Quase todo o código é lido da esquerda para a direita e de cima para baixo. Cada linha representa uma expressão ou cláusula, e cada grupo de linhas representa um pensamento completo. E para facilitar a leitura, esses pensamentos devem ser separados por linhas em branco (pular linha).

Não deixe o código se estender demais em uma linha só, procure seguir o tamanha do ambiente de desenvolvimento (IDE) que está utilizando para saber até onde ir. Não ultrapasse a margem!

Indentação
O código-fonte possui hierarquias, os níveis podem ser definidos por:

  • Classes
  • Métodos/Funções
  • Blocos de códgigo

Mas esses níveis não são facilmente identificados em um código com texto escrito no mesmo alinhamento. Para solucionar isso utiliza-se um recurso chamado Indentação, que nada mais é do que o afastamento do texto da sua margem, possibilitando a visualização dos níveis citados.
Sem indentação códigos tornam-se ilegíveis para humanos.

Observe a diferença de um código sem nenhuma formatação (Trecho de Código - 08) de um formatado corretamente (Trecho de Código - 09): 

            
                // sem separação de linhas e sem indentação
                public class Pessoa {
                    private String nome;
                    private int idade;
                    private String cpf;
                    public Pessoa(String nome, int idade, String cpf) {
                        this.nome = nome;
                        this.idade = idade;
                        this.cpf = cpf;
                    }
                    public String mostrarFasesPessoa(Pessoa pessoa) {
                        if(pessoa.getIdade() <= 13) {
                            return pessoa.getNome() + " é criança.";
                        }
                        else if(pessoa.getIdade() < 18) {
                            return pessoa.getNome() + " é adolescente.";
                        }
                        else if(pessoa.getIdade() < 60) {
                            return pessoa.getNome() + " é adulto.";
                        }
                        pessoa.getNome() + " é idoso.";
                    }
                }
Trecho de Código - 08 
            
                // formatado corretamente
                public class Pessoa {
                    private String nome;
                    private int idade;
                    private String cpf;

                    public Pessoa(String nome, int idade, String cpf) {
                        this.nome = nome;
                        this.idade = idade;
                        this.cpf = cpf;
                    }

                    public String mostrarFasesPessoa(Pessoa pessoa) {
                        if(pessoa.getIdade() <= 13) {
                            return pessoa.getNome() + " é criança.";
                        }
                        else if(pessoa.getIdade() < 18) {
                            return pessoa.getNome() + " é adolescente.";
                        }
                        else if(pessoa.getIdade() < 60) {
                            return pessoa.getNome() + " é adulto.";
                        }
                        
                        pessoa.getNome() + " é idoso.";
                    }
                }
Trecho de Código - 09

Comentários

Não existe nada mais útil do que um bom comentário. Porém, não existe nada pior do que um comentário ruim, redundante e desnecessário.

Os comentários servem para quando não conseguimos expressar algo somente com os nomes presentes no código.

Claro que você pode fazer comentários nos seus projetos pessoais, mas o recomendável é não termos que encher os códigos de comentários. 

            
                // verifica se o funcionario pode receber benefícios
                if(funcionario.getTempoTrabalho > 20 && funcionario.getIdade() > 65) {

                }
Trecho de Código - 10

Agora vamos ver como esse código poderia ficar para que não houvesse a necessidade de ter comentários acima dele. 

            
                if(funcionario.podeReceberBeneficios()) {

                }
Trecho de Código - 11

No Trecho de Código - 11 o método foi adaptado para que ficasse mais clara a sua função e não precisasse ter um comentário explicando o que ele faz.

Vamos ver mais alguns exemplos de comentários desnecessários: 

            
                /* Nome da pessoa */
                private String nome;

                /* Dia do mês */
                private int diaDoMes;

                /* Construtor padrão */
                public Pessoa() {
                }

                /* Retorna a idade */
                public int getIdade() {
                    return this.idade;
                }
Trecho de Código - 12

Além dos comentários explicativos têm também os códigos que não são mais usados e são comentados para não interfirirem na execução do programa. Se ver que um código está comentado e não vai ser usado, apague-o sem pena!

Mais algumas dicas

  • Use nomes pronunciáveis
  • Não use abreviações ou códigos em nomes
  • Não faça trocadilhos em métodos ou variáveis
  • Evite repetição de código

Por último, uma regrinha que é mostrada no livro Código Limpo que todo programador deveria seguir:

Regra do escoteiro
"Deixe sempre o acampamento mais limpo do que quando você o encontrou!"
O mesmo vale para nosso código. Devolva sempre o código melhor do que você o obteve. Se todo desenvolvedor no time tiver esta visão, e devolver um pedacinho de código melhor do que estava antes, em pouco temos teremos uma grande mudança.

Se gostou das dicas e quiser se aprofundar mais nas boas práticas de programação, recomendamos ler o livro que serviu de base para este guia (mencionado na página inicial).
