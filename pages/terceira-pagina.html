Métodos ou Funções


Assim como as variáveis, os métodos ou funções devem seguir um padrão de nomenclatura e devem descrever de forma clara o que fazem no código.
É recomendável que os nomes dos métodos sejam verbos, para que representem ações.

No Trecho de código - 04 temos um exemplo de método simples em um código Java: 

            
                public int somar(int primeiroValor, int segundoValor) {
                    return primeiroValor + segundoValor;
                }
Trecho de Código - 04

Não se apresse ao escolher um nome para seus métodos. Certifique-se de que ele seja descritivo e revele sua função ao lê-lo.

É provável que nomes grandes em códigos passem uma má impressão a primeira vista, mas os nomes nos códigos são 90% responsáveis pela legibilidade do software.

É preciso escolher sabiamente seus nomes para manter os métodos claros e relevantes. Nomes são muito importantes para serem tratados de qualquer jeito!

Vejamos mais alguns exemplos. 

            
                1  public String x(List<double> y) {
                2      double q = 0.0;
                3 
                4      for(int kk = 0; kk < 3; kk++) {
                5          q = q + y.get(kk);
                6      }
                7
                8      q = q / 3;
                9
                10     if(q >= 7.0) {
                11         return "Aprovado";
                12     }
                13    
                14     return "Reprovado";
                15 }
Trecho de Código - 05

Observe o Trecho de Código - 05 acima, você consegue entender sem dificuldade o que ele faz?
Eu tenho certeza que a resposta é NÃO. Mesmo que você não saiba ou nunca tenha visto essa sintaxe, ela não é o motivo de você ter dificuldade de entender o que o método faz. Da forma que ele está é apenas um emaranhando de símbolos e números mágicos.

Vamos ver esse mesmo método escrito utilizando as boas práticas para um código limpo. 

            
                1  public String mostrarResultadoFinalDoAluno(List<double> notas) {
                2      double media = 0.0;
                3 
                4      for(int contador = 0; contador < notas.size(); contador++) {
                5          media = media + notas.get(contador);
                6      }
                7
                8      media = media / notas.size();
                9
                10     if(media >= 7.0) {
                11         return "Aprovado";
                12     }
                13
                14     return "Reprovado";
                15 }
Trecho de Código - 06

Analisando o método agora podemos perceber a diferença que os nomes fazem, não acha?
Apesar de parecer complexo, basicamente o que ele está fazendo é exatamente o que está dizendo no seu nome "mostrar o resultado final do aluno", onde é um método que recebe as notas de um aluno, soma essas notas uma por uma, com a soma armazenada na variável "media", é calculada a média das notas (soma das notas dividida pelo número total de notas).

Após isso, a função condicional verifica se essa média é maior que 7, em caso verdadeiro é retornado que o aluno foi aprovado, caso contrário ele não entra na verificação e retorna diretamente que o aluno foi reprovado.

Uma outra orientação é: Métodos/funções devem fazer uma coisa só!

Costuma ser tentador criar métodos que tenham várias funcionalidades dentro deles, mas isso não é o recomendado, porque além de deixar o método extenso, dificulta sua manutenção se for preciso refatorá-lo .
Uma alternativa para essa situação é, se você precisar fazer mais de uma operação, divida a função criando outro método, e caso ainda precise, chame-o dentro do método principal. 

            
                1  // método principal
                2  public String gerarTituloEleitor(Pessoa pessoa) {
                3 
                4      if(temIdadePermitida(pessoa.getIdade())) {
                5
                6          verificarDocumentos(pessoa.getDocumentos());
                7
                8          // aqui fica o código que gera o título...
                9
                10         return "Título gerado com sucesso.";
                11     }
                12    
                13     return "Você ainda não tem idade para gerar o título de eleitor." 
                14 }
                15
                16 // primeiro método auxiliar
                17 public boolean temIdadePermitida(int idade) {
                18     if(idade >= 16) {
                19         return true;
                20     }
                21
                22     return false;
                23 }
                24
                25 // segundo método auxiliar
                26 public boolean verificarDocumentos(List<String> documentos) {
                27     // código para verificar os documentos...
                28 }
Trecho de Código - 07

Note que poderíamos ter no método principal tudo o que os métodos auxiliares estão fazendo, mas decidimos deixar no principal somente o que é descrito pelo seu nome, gerar o título. O tratamento dos documentos, a verificação de idade e qualquer outra função para que o título possa ser gerado, pode ficar em um método separado.

Isso ajuda a deixar o código organizado e mais fácil de entender. Além de que esses métodos auxiliares podem ser utilizados por outros métodos, não só o principal, o que reduz também a repetição de código.
