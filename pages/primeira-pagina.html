Números mágicos


Números mágicos são aqueles números aleatórios que de vez em quando você acaba encontrando no código, eles são “mágicos” porque se assemelham com o truque de um mágico, sendo necessário uma explicação para que sejam entendidos. 

            
                1 // método que recebe dois números e soma-os
                2 metodoDeSoma(int primeiroValor, int segundoValor) {
                3     soma = primeiroValor + 2;
                4     return soma;
                5 }
Trecho de Código - 02

No Trecho de Código - 02, de onde veio o número 2? Qual seu significado? Pois é…
Imagina ter números assim espalhados por códigos gigantes em um sistema. E, por incrível que pareça, acontece com frequência.
Ter estes números no seu código abre portas para bugs e problemas mais sérios na aplicação, por exemplo:

  • Dificultar a leitura do código
  • Prejudicar a manutenção do código

Para evitar os números mágicos, sempre que possível utilize constantes para representar esses valores.
As constantes são variáveis com o propósito principal de dar significado a um valor que não precisa sofrer modificações com frequência. 

            
                1 // método que recebe dois números e soma-os com 
                2 // o ano em que estamos
                3 int ANO_ATUAL = 0;
                4 metodoDeSoma(int primeiroValor, int segundoValor) {
                5    soma = primeiroValor + segundoValor + ANO_ATUAL;
                6    return soma;
                7 }
Trecho de Código - 03

No Trecho de Código - 03 podemos ver que não há mais nenhum número solto.
Temos uma constante ANO_ATUAL que está guardando um valor que pode ser alterado.
Então se o ano estivesse sendo usado em várias partes de um código, só seria necessário modificá-lo na constante, facilitando a manutenção desse código.
Se o valor do ano fosse um número mágico, seria necessário encontrar todas as suas ocorrências para conseguir alterá-lo e isso muitas vezes é trabalhoso!
